1 · The Anatomy of a DataFrame

A DataFrame is a 2-D table with three labelled parts: the index (row labels), the columns (column labels), and the values. Every column is a Series; the DataFrame is what you get when you bundle Series together that share the same index. Hover any cell, header, or index label to see what it is.

Try hovering each part. The blue badge under each column header is the dtype — pandas tracks the type of every column so it knows how to do math.

Toggle view

A DataFrame is literally a dictionary where each value is a Series — and every Series shares the same index. This view makes that explicit.

Three things to remember

  • Index — row labels. Doesn't have to be 0,1,2 — can be dates, IDs, names.
  • Columns — column labels. Each is also a Series's .name.
  • dtypes — per-column. Mixed types fall back to object — slower.

2 · loc vs iloc — the headline concept

The single biggest source of confusion for pandas beginners. Both panels show the same DataFrame. Click an example or type your own expression on either side and watch the same data respond differently.

🏠 loc — by label (string row name, column name) 📍 iloc — by integer position (0-based)

🏠 .loc — label based

📍 .iloc — position based

⚠️ The slicing gotcha — click both and compare

.loc includes the right-hand label (3 rows back). .iloc follows Python list semantics — the right-hand index is excluded (2 rows back). This catches everyone, every time.

3 · Boolean Masking — the bouncer's checklist

Filtering rows in pandas is a two-step process: compute a True/False mask, then apply the mask. Each row's True/False decides whether it survives. Slide the threshold to watch it happen.

Notice: the function knows nothing about pandas — it's just a function from one value to one value. .apply() is the loop that calls it for every element. That's why apply() is slow on huge data — it's still a Python loop. Use vectorized arithmetic when you can.

5 · sort_values & rank — ordering with ties

Sorting reorders rows. Ranking labels rows in order — without moving them. Notice this leaderboard has a deliberate tie (Bob and Eve both scored 78). The method= argument decides how the tie is resolved.

Leaderboard

How each method handles ties

method tie behaviour
averagetied rows share the average rank (e.g. 2.5, 2.5)
minall tied get the lowest rank in the group (2, 2, then next is 4)
maxall tied get the highest rank (3, 3, then next is 4)
denselike min, but the next rank is +1 (2, 2, then next is 3)
firstbreak ties by order in the data (2, 3 — whoever appeared first wins)

Sales leaderboards usually want min (no shared 2.5th place). Olympic medal counts usually want min too (two golds → no silver awarded).
